Think of this file as a template in a traditianal framework monolith. in Symfony, it's index.twig.php, in Laravel, it's index.blade.php, in Magento, it's index.phtml â€¦ you get the picture. The only requirements with this idea:

  1. Any template that includes JSX root nodes must include `app.js` at the bottom of the page.
  2. JSX nodes can't (currently) be nested (this is subject to change with a more robust DOM crawler)

TL;DR: this works by looping through each of the available React components created. For each of these components, a `document.getElementsByTagName` is called to see if a tag in the DOM matches the component. If the tag exists, a React root is created, the tag/Component is injected into the root, and React renders the results.
